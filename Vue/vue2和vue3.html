    # vue2和vue3

    # 1. 为什么vue3中定义响应式数据要分别用ref()和reactive()来定义，显得更复杂了。vue2中统一定义在data(){}中，它不香吗？

    个人理解：

    第一点，效率

    Vue2-------所有数据都遍历绑定getter和setter

    Vue3-------提升效率-------选择性绑定getter和setter------所以有了ref()

    第二点，解决vue2添加和删除，界面不更新的小bug

    Vue2------对于复杂数据类型，后添加删除的属性（指按下标模式添加或删除的）未绑定getter和setter-------vue2不能监测到------界面不更新

    Vue3------为了解决vue2不监听添加删除属性的问题-------有了reactive()--------底层采用代理Proxy-----Proxy可以拦截对象中任意属性的变化。

    # 2. 既然Proxy这么好，为什么底层不统一都用Proxy？而是ref()用Object.defineProperty()延续vue2的操作，reactive()用代理Proxy？

    个人理解：

    因为普通数据不具有深层次结构，使用简单的setter和getter就足以实现响应式。对于普通数据使用Proxy就显得累赘了。

    # 3. 为什么vue2的data(){}和vue3的ref()底层都是用的Object.defineProperty()，但vue3的ref()后的数据要加上.value才能使用？

    个人理解：

    因为ref()函数返回的是一个RefImpl{…}对象，我们定义的值存在对象里的value中。而vue2中的数据经过数据代理挂载在VueComponent实例中，所以使用时直接this.xxx就可以了。

    # 4. 为什么Proxy代理中返回值要用 反射Reflect实现对源对象的操作？直接操作不是更简单吗？

    个人理解：对于框架底层编码来说，用Reflect更方便捕获错误，可以少写一些try…catch。
